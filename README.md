# HarmonyOS 2048 小游戏应用

本项目基于开源 HarmonyOS 2048 示例工程进行二次开发，面向开源软件课程作业，完成了多端响应式适配、数据接续、多种游戏模式、成就与统计系统、键盘控制以及整体视觉升级。

项目不仅保留了 2048 的基础玩法，还围绕 HarmonyOS 的“一次开发，多端部署”和“数据自由流转”能力进行了扩展，使同一套工程能够适配手机、平板和 2in1 设备，并支持游戏状态的保存、恢复与应用接续。

---

## 项目亮点

- 同一套代码适配手机、平板和 2in1
- 手机端纵向布局，宽屏端棋盘与控制区左右分栏
- 经典、限时、步数挑战三种游戏模式
- 一步撤销、模拟快照保存与恢复
- UIAbility 应用接续代码与配置
- 成就系统、数据统计和最近游戏记录
- 方向键与 W/A/S/D 键盘控制
- 统一的胜利、失败、时间到和挑战结束结算界面
- 浅色卡片化视觉设计与响应式首页

## 功能说明

### 1. 基础游戏功能

项目保留并完善了 2048 的核心规则：

- 上、下、左、右移动数字方块
- 相同数字相遇时合并
- 合并后累计分数
- 自动生成新的数字方块
- 无合法移动时判定本局结束
- 合成 2048 时触发胜利反馈

此外新增：

- 当前分数显示
- 各模式最高分
- 有效步数统计
- 游戏说明
- 新游戏
- 一步撤销
- 统一结算界面

无效滑动不会增加步数，也不会消耗步数挑战中的剩余次数。

### 2. 三种游戏模式

#### 经典模式

无时间和步数限制，按照标准 2048 规则持续游戏，直到合成 2048 或棋盘无法继续移动。

#### 限时挑战

在 90 秒内获得尽可能高的分数。倒计时结束后，系统停止游戏并显示本局结算结果。

#### 步数挑战

在限定的有效移动次数内获得尽可能高的分数。只有真正改变棋盘状态的移动才会消耗次数。

### 3. 一步撤销

每次有效移动前，系统保存移动前的游戏状态。点击“撤销一步”后，可恢复：

- 棋盘格数据
- 当前分数
- 当前步数
- 最高分
- 胜负状态
- 挑战模式相关状态

撤销完成后会清空本次撤销记录，避免无限连续撤销。

### 4. 成就系统

项目加入了成就检测、解锁提示和持久化保存。成就包括：

- 初出茅庐：首次合成 32
- 小有成就：首次合成 128
- 数字大师：首次合成 512
- 最终目标：首次合成 2048
- 效率玩家：在较少步数内合成指定数字
- 限时高手：在限时模式达到指定分数
- 坚持不懈：累计完成指定数量的游戏

新成就解锁后会显示提示卡片，已解锁状态和解锁时间会保存到本地。

### 5. 游戏数据统计

应用记录并展示多项游戏数据，包括：

- 总游戏局数
- 各模式游戏局数
- 胜利次数与失败次数
- 历史最高分
- 历史最大合成数字
- 累计有效移动步数
- 特定数字的最少达成步数
- 最近游戏时间
- 最近 5 局游戏记录

统计数据使用本地持久化存储，重新进入应用后仍可查看。

### 6. 多种操作方式

在手机和平板上，可以通过触摸滑动控制棋盘。

在平板、2in1 或连接键盘的设备上，还支持：

| 操作 | 按键 |
|---|---|
| 向左移动 | ← 或 A |
| 向右移动 | → 或 D |
| 向上移动 | ↑ 或 W |
| 向下移动 | ↓ 或 S |

键盘操作和触摸滑动复用同一套移动逻辑，保证计分、计步和模式规则一致。

## 一次开发，多端部署

项目使用同一套代码适配不同设备形态，并以页面有效宽度作为布局依据。

### 手机端

- 首页游戏卡片纵向排列
- 2048 页面采用纵向布局
- 统计信息、操作按钮和棋盘按顺序排列
- 页面高度不足时支持滚动
- 棋盘始终保持正方形

### 平板与 2in1

- 首页游戏卡片横向或网格排列
- 2048 页面采用左右分栏
- 左侧显示正方形棋盘
- 右侧显示模式、统计、操作和辅助入口
- 支持方向键及 W/A/S/D 操作

项目以约 600vp 作为窄屏与宽屏布局切换断点。窗口尺寸变化后，页面会自动调整布局。

## 数据自由流转

### 游戏快照

项目将当前游戏状态整理为可序列化的普通数据快照。快照包含：

- 16 个棋盘格数值
- 当前分数
- 当前步数
- 最高分
- 胜利与失败状态
- 当前游戏模式
- 限时或步数挑战的剩余状态

项目不直接传输 `Board` 或 `Tile` 类实例，而是传输普通数据，并在恢复时重新构建棋盘对象。

### 模拟接续

应用提供“保存模拟快照”和“恢复模拟快照”功能，用于单设备验证完整的数据链路：

1. 保存当前游戏状态
2. 继续移动或开始新游戏
3. 恢复此前快照
4. 还原棋盘、分数、步数和模式状态

### UIAbility 应用接续

工程已配置并实现应用接续相关能力，包括：

- `continuable`
- `continueType`
- `UIAbility.onContinue`
- `onNewWant`
- `LaunchReason.CONTINUATION`
- `Want.parameters`

`GameStateStore` 用于在页面组件与 `EntryAbility` 之间传递游戏快照。

> 说明：系统级跨设备接续通常需要两台满足条件的 HarmonyOS 设备、可信设备环境及相应系统能力。本项目已完成接续代码和配置，并通过本地模拟快照验证数据序列化与恢复流程；未将单设备模拟恢复表述为真机跨设备实测。

## 界面与交互设计

项目对原始界面进行了整体产品化改造：

- 使用柔和浅色背景和统一主色
- 游戏入口采用卡片化设计
- 统计信息采用独立数据卡片
- 主要操作与辅助操作具有不同视觉层级
- 手机和平板采用不同的空间组织方式
- 模式选择、成就、统计和游戏说明使用统一风格
- 胜利、失败、时间到和挑战结束使用统一结算遮罩
- 按钮、卡片、间距与圆角保持一致

## 运行环境

建议使用与工程配置兼容的 DevEco Studio 和 HarmonyOS/OpenHarmony SDK。

SDK 与 API 版本以项目中的以下配置为准：

```text
harmonyos-games/build-profile.json5
harmonyos-games/entry/src/main/module.json5
```

---

## 运行方法

1. 使用 DevEco Studio 打开：

   ```text
   Game2048-Homework/harmonyos-games
   ```

2. 等待工程同步和依赖安装完成。

3. 在设备管理器中启动手机、平板或 2in1 模拟器。

4. 选择 `entry` 运行配置和目标设备。

5. 点击 Run，安装并启动应用。

如果模拟器中仍保留旧版本应用，建议先卸载旧应用，再重新构建和运行。
